GUI简介

  • GUI 是一个web前端开发框架的开源项目，目的是创建一个优雅、跨浏览器、直观、强大的web ui framework，从而加快web前端开发，使得开发更加容易。
  • 使用了Less、Nodejs、Bower、Qunit、Normalize等， 借鉴了Bootstrap的架构思想；各个组件依赖于Jquery，以Jquery Plugin的形式开发。
  • 使用Git来实现版本控制，使用Github来管理项目、协作开发。 项目地址：https://github.com/guoyao/gui

项目整体结构

  • assets目录存放项目需要的image、icon等资源文件。
  • bower_components、demo、dist、
    node_modules 都是临时目录，每次编译时都会自动生成。分别存放项目依赖、demo生成、编译的最终结果、自动化编译/测试等所需的nodejs模块。
  • demo_template目录存放demo模板文件、文档等。
  • js目录存放项目所有的javascript文件，在编译时会被合并。

项目整体结构

  • .travis.yml持续集成脚本。
  • less目录存放项目所有的less文件，编译时会先合并，然后翻译成css。
  • bower.json声明和管理项目所需要的第三方依赖，包括jquery、qunit、requirejs等。
  • Gruntfile.js项目自动化编译、合并、测试、打包、demo生成等脚本。
  • package.json项目的描述文件，所需的nodejs模块依赖等。

Less简介

Less是一种动态样式语言，属于CSS预处理语言的一种，它使用类似CSS的语法，为CSS的赋予了动态语言的特性，如变量、继承、运算、函数等，更方便CSS的编写和维护。

Less可以在多种语言、环境中使用，包括浏览器端、桌面客户端、服务端。

Node.js简介

Node.js是一个Javascript的运行环境，一个可以快速构建网络服务及应用的平台；采用C++语言编写而成， 基于Chrome's JavaScript runtime，对Google V8引擎进行了封装，性能很好。

Node.js使用 事件驱动，非阻塞I/O模型（异步）， 使得它轻量级，高效，非常适合处理并发请求，因此构建在Node上的服务相比其他技术实现的服务表现要好得多； 完美的适用于分布式的数据密集型实时应用。

Node.js提供了很多系统级的API，如 文件操作、网络编程 等。 浏览器端的Javascript代码在运行时会受到各种安全性的限制，对客户系统的操作有限。相比之下，Node.js则是一个全面的后台运行时， 为Javascript提供了其他语言能够实现的许多功能。

Node.js简介

  • 特点
    • 纯event IO并发，极其适合超高IO并发场景
    • JavaScript V8引擎的性能非常好
    • 仅适合IO密集型并发，不适合CPU密集并发
    • 不适合开发业务逻辑过于复杂的应用
    • 异步编程复杂度和调试难度偏高
  • 应用场景
    • 游戏服务器端
    • 长连接服务器端, 单机往往维持几千上万长连接的
    • 对IO并发要求极高的情况
  • 问题
    • event IO异步编程是一种相当原始的编程方式，IO操作都需要异步回调，稍微复杂的代码有十几层嵌套，代码可阅读性和维护性很差。

Bower简介

Bower 是 twitter 推出的一款包管理工具，基于node.js的模块化思想，把功能分散到各个模块中，让模块和模块之间存在联系，通过 Bower 来管理模块间的这种联系。

包管理工具一般有以下的功能：

  • 注册机制：每个包需要确定一个唯一的 ID 使得搜索和下载的时候能够正确匹配，所以包管理工具需要维护注册信息，可以依赖其他平台。
  • 文件存储：确定文件存放的位置，下载的时候可以找到，当然这个地址在网络上是可访问的。
  • 上传下载：这是工具的主要功能，能提高包使用的便利性。比如想用 jquery 只需要 install 一下就可以了，不用到处找下载。上传并不是必备的，根据文件存储的位置而定，但需要有一定的机制保障。

Bower简介

  • 依赖分析：这也是包管理工具主要解决的问题之一，既然包之间是有联系的，那么下载的时候就需要处理他们之间的依赖。下载一个包的时候也需要下载依赖的包。

通过 Bower 来安装GUI: bower install gui

通过 bower search 命令来查找，如：bower search gui

通过 bower info 命令来查看package的信息，如：bower info gui

Grunt简介

Grunt是一个基于Node.js的自动化构建工具。

Java世界里的Maven提供了强大的包依赖管理和构建生命周期管理。在JavaScript的世界里， 随着Node.js的流行，JavaScript原生的构建工具已经成为可能。

Grunt是基于Node.js的自动化任务运行器。Grunt结合NPM的包依赖管理，完全可以媲美Maven。 Grunt天然适合前端应用程序的构建，不仅限于JavaScript项目，同样可以用于其他语言的应用程序构建。 越来越多的JavaScript项目已经在使用Grunt，其中包括著名的 jQuery、Bootstrap、WordPress、 Backbone、Pure、Modernizr等。

几点建议

  • 代码提交前必须通过自动化代码质量检查（目前基于 jshint），杜绝任何错误、警告。
  • 重要代码必须写Unit Test（目前基于 QUnit），有了单元测试，代码重构就基本无后顾之忧了。
  • 单元测试的代码覆盖率至少要超过60%（目前基于 coveralls）。覆盖率太低的测试没有实际意义。
  • 多阅读著名开源项目的代码，这些代码都经过千锤百炼，都是精华中的精华。坚持code review（代码审查），查看别人的代码，取长补短。
  • 坚持代码重构，这样才能写出更优秀，更易扩展，更易维护的代码。
